> **Round-5 engine-reality grounding (decisive — verified against the repo, not just mined text):** SwarmUI's **native ACE-Step 1.5 is TEXT-TO-MUSIC ONLY** — it exposes only Audio Style + Audio Duration (+ bpm), exactly the `textaudiostyle`/`textaudiobpm`/`textaudioduration` the repo wires. The ACE-Step *model* supports repaint, extend (context-length), lyric/style range-edit, and audio2audio — but **only via ComfyUI custom workflows that must INGEST an existing audio file**, which hits the repo's twice-documented **base64-injection dead-end** (the same wall that killed custom I2V and blocks S2V/Wan2.2-Animate). The recurring mined claim "ACE-Step natively supports repaint, so it's a pure-UX win" is therefore **wrong against this install**: the model can, the SwarmUI loader path cannot receive the audio. This splits the audio-editing features sharply — DSP/no-model ops and LLM-orchestration ship now; every ACE-Step-NATIVE edit op (seamless repaint/extend/lyric-edit/audio2audio) is **DROPPED/model-gated** behind a native SwarmUI audio-init path. Foley = **HunyuanVideo-Foley** (the `WanFoley.json` custom workflow), which runs *because the audio is generated in-graph*, and is **video-conditioned** (no text-only SFX surface exists today). TTS = **Chatterbox :8004** (`/tts`, `/v1/audio/speech`, `/upload_reference`; exposes `exaggeration`+`cfg_weight`; **no `/vc` endpoint** despite `vc.py` weights on disk).
