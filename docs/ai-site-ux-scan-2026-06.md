# DokiDex Consumer-AI Platform Scan: UX Patterns Worth Stealing

DokiDex is a fully-local, uncensored, single-window AI web studio running on a 32GB RTX 5090 over SwarmUI/ComfyUI for media plus a local llama.cpp (with llama-swap) for text. Its capability surface — image, video, lip-sync, audio, voice, and an OpenAI-compatible local LLM — already rivals or matches what most consumer platforms ship; its one genuine gap is the **conversational/assistant surface** and the **consumer-friendly wrapping** of raw model power. This report dedupes 77 scouted platforms down to **75 unique** (collapsing the two cross-category repeats, Nomi.ai and Glif) and reads each through one lens: what consumer UX move does it pioneer, and can DokiDex reproduce that move locally over SwarmUI + llama.cpp? The recurring finding is that almost every signature UX here is an *integration/front-end* play, not a model play — DokiDex usually already owns the hard part (the engine) and is missing only the polished surface. The notable exceptions are proprietary models whose *quality* (not concept) is out of local reach: Reve's "Halfmoon," ByteDance Seedream, Meta Emu, Inflection's voice, and the realtime-avatar renderers (Tavus Phoenix-4, Hedra Character-3). Many companion and image platforms also hide their backing chat LLM entirely — flagged below as proprietary-but-unknown, most likely fine-tuned open Llama/Mistral-class weights.

## Top consumer-UX patterns to adopt in DokiDex

Ordered by leverage for a local SwarmUI + llama.cpp single-user studio. Tags: **[feasible-locally]** = buildable today with the current stack; **[gated-on-optional-install]** = needs an additional local model/component (Whisper, Kokoro, RVC, LatentSync, a VLM/coder GGUF, an embedding model) DokiDex can fetch on demand; **[upstream-blocked]** = the *concept* is local but matching the named platform's *quality* depends on a proprietary closed model.

- **Inline generated media as chat messages (Botify/Imagine/Yodayo).** Let the LLM emit a tool-call when a visual fits, fire a SwarmUI image or Wan2.2 clip for the active persona, and render it as an inline bubble. This is the single highest-leverage move: it uses DokiDex's strongest asset (media gen) to fix its one real weakness (the conversational surface). **[feasible-locally]**
- **Bind a generated character to a chat persona — one object for appearance + identity + brain (Candy/Yodayo/SoulGen).** Persist a per-character JSON: appearance tags + an IPAdapter/LoRA face reference + a persona system-prompt + a chosen llama-swap model. Every photo request re-pins the same face; every chat uses the same brain. This unifies DokiDex's two halves and unlocks most companion UX. **[feasible-locally]**
- **A persistent, readable/editable memory layer via local RAG (Nomi/Kindroid).** Structure memory as tiers — a static backstory file + a vector store of "key memories" retrieved into context + a periodic LLM-summarized journal. Recall across weeks without giant context windows; the user can read and edit it. Requires a local embedding model. **[gated-on-optional-install]**
- **Per-conversation/per-role model hot-swap surfaced as a dropdown (SpicyChat/Chub/Pollo/Continue).** Expose installed local models (and saved workflows) as a visible picker so the user swaps the backing brain or engine on the same persona/prompt mid-session, with a side-by-side compare grid. llama-swap already does the hard part; this is the unifying shell pattern Venice/Pollo are admired for. **[feasible-locally]**
- **Import the existing character-card ecosystem (Chub/SpicyChat/Venus).** Add a TavernAI/Chub PNG-embedded card importer. Because DokiDex's LLM is OpenAI-compatible, this instantly makes tens of thousands of community personas usable locally and free — enormous content leverage for near-zero model work. **[feasible-locally]**
- **Save ComfyUI/SwarmUI workflows as one-input "recipe/glif" cards, with duplicate-and-tweak (Glif/HF Spaces/getimg Elements).** Hide the node graph; surface a few inputs as a card; auto-generate the form from a model/workflow's declared inputs (the Gradio/Spaces lesson). Turns raw pipeline power into casual, remixable consumer UX. **[feasible-locally]**
- **Tag/chip composers and named style tiles instead of a raw prompt box (PornPen/DreamGF/Promptchan/DeepAI).** A curated tag palette and style-tile strip that compile to a SwarmUI prompt + the right checkpoint/LoRA behind the scenes. Ships casual approachability on day one over engines DokiDex already runs, alongside the expert prompt box. **[feasible-locally]**
- **Character/identity consistency as a first-class productized loop (SoulGen/PixAI/getimg).** "Lock my OC, re-pose it" via a trained LoRA or IPAdapter/PuLID embedding reused on every render, with optional in-app LoRA training. SwarmUI/ComfyUI already support every piece; this is the feature DokiDex is best positioned to win because it owns the full local pipeline. **[gated-on-optional-install]** (LoRA training/IPAdapter assets)
- **Editable-script = editable-media, re-rendering only dirty segments (Synthesia/Wondercraft/Descript/Vozo).** Store per-segment (text, audio, video) triples; when text changes, silently re-run only TTS + lip-sync for that segment. The load-bearing binding behind every avatar/podcast/dubbing studio. Needs Whisper + Kokoro/voice-clone TTS + a lip-sync model. **[gated-on-optional-install]**
- **A checkpoint-everything Plan/Act agent panel (Cline/Aider/Replit).** A coding/agent surface over llama.cpp with a Plan/Act toggle, per-step human approval, "AI edit = shadow-git commit," a repo-map for context, and (high value) a headless-Playwright self-verify loop. Cline/Aider are open, already target local endpoints, and are near-liftable references. **[gated-on-optional-install]** (a strong coder GGUF)
- **One-step "photo + audio → talking clip" and "/imagine in chat → image" collapses (Hedra/D-ID/Imagine).** Replace node graphs with a single card or a chat intent. D-ID/Hedra are essentially the InfiniteTalk pipeline DokiDex already runs; the win is purely UX collapse. Talking-clip needs a lip-sync model; the chat-image intent is pure routing. **[gated-on-optional-install]** (lip-sync) / **[feasible-locally]** (image intent)
- **Assistant produces structured artifacts, not just prose — diagrams, mind maps, Sparkpages (Napkin/Felo/Genspark).** Have the local LLM emit diagram/graph JSON (Mermaid/flow/mind-map) or a multi-section page template in the same pass as its answer, rendered to editable SVG/HTML. Fast, deterministic-ish locally, and it complements DokiDex's visual strength. **[feasible-locally]**
- **Empathetic-voice / suggested-reply-chips conversational polish (Pi).** A warm, brief, one-question-at-a-time system prompt + Kokoro TTS + LLM-generated follow-up chips so the user never faces a blank box. Cheap, high-perceived-quality; only the bespoke voice timbre is a gap. **[gated-on-optional-install]** (TTS) / **[upstream-blocked]** for matching Inflection's exact voice quality.
- **Realtime conversational avatar / voice-call mode (Tavus/Hedra Live/Muah).** Stream llama.cpp → Kokoro → LatentSync into a WebRTC preview. The concept is local; 40fps photoreal rendering and sub-100ms barge-in turn-taking on one 5090 are the ceiling, so a few-fps "good enough" avatar call is the realistic local target — still a strong differentiator. **[upstream-blocked]** (fidelity/latency vs Phoenix-4/Character-3).
